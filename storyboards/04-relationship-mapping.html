Relationship Mapping

Define guest relationships using the visual matrix editor. Specify who should sit together (partners, family, friends) and who should be kept apart for optimal seating arrangements.

5 steps ~3 min to complete Prerequisite: Guests added
1

Navigate to Relationship View

Go to the Guests tab and click the Relationships toggle button to switch from the list view to the relationship matrix.

2

Understand the Matrix

The matrix shows all guests on both axes. Each cell represents the relationship between two guests. Colors indicate relationship types:

  • Pink: Partner/Spouse (highest priority)
  • Purple: Family member
  • Blue: Friend
  • Green: Colleague
  • Red: Avoid (keep apart)

Set a Relationship

Click any cell to open the relationship selector. Choose the type that best describes how these two guests relate. The cell updates immediately with the corresponding color.

Use Bidirectional Mode

Enable Bidirectional mode to automatically mirror relationships. When you set "Sarah → John = Partner", it also sets "John → Sarah = Partner". This saves time for symmetric relationships.

Tip: Bidirectional mode is on by default. Turn it off if you need asymmetric relationships (rare, but useful for one-way preferences).

Review and Optimize

Once relationships are defined, go to the Optimize tab to run the seating algorithm. The optimizer uses your relationship data to maximize positive pairings and respect "avoid" constraints.
